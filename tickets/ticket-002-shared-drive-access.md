# Ticket 002: Shared Drive Access Issue

## Issue
User could not access a department shared drive.

## Symptoms
- "Access Denied" message
- Shared drive not opening
- Other users had access

## Troubleshooting Steps
1. Verified network connectivity
2. Confirmed drive path
3. Checked user group membership
4. Reviewed permissions
5. Re-added correct access group
6. Had user log out and back in

## Resolution
Access restored after updating permissions.

## Root Cause
User was missing the required security group membership.
